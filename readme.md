🏠 家具商城+智能家居控制+社交一体化项目（前端）

> 初学者 Vue3 练手项目 😊 前后端分离架构的前端实现，网页设计基于个人理解与想象完成，功能仍在迭代打磨，欢迎各位前辈、小伙伴提建议、指正问题~

一、项目介绍 📖

本项目是「家具商城+智能家居控制+社交一体化」全栈项目的前端部分，负责用户交互界面的实现与后端服务的对接。核心覆盖三大生活化场景的前端展示与交互逻辑，所有页面设计均为个人自主构思，样式参考多个优质资源打磨而成：

- 家具商城 🛋️：实现商品列表展示、详情查看、购物车操作、简单下单流程等交互界面；

- 智能家居控制 🏡：提供智能设备列表、设备状态展示、模拟控制指令发送等可视化界面；

- 社交互动 🗣️：实现用户关注、动态发布与展示等社交功能的前端交互。

当前项目处于初期开发阶段，界面美观度和交互流畅度仍有优化空间，后续会持续迭代完善~

三、技术栈选型 🛠️

（初学者友好型技术组合，以易上手、实用性强为核心原则）

- 核心框架：Vue 3.js（Composition API 风格）

- 路由管理：Vue Router 4

- 状态管理：Pinia（轻量易用，替代 Vuex）

- UI 组件库：Element Plus（基础组件快速搭建）

- HTTP 请求：Axios（对接后端 API 接口）

- 样式补充：UIverse（优质自定义组件参考）、原生 CSS（个人定制样式）

- 开发工具：Visual Studio Code + Git

四、样式设计来源 🎨

本项目所有网页设计均基于个人对产品的理解与想象自主构思，样式实现过程中参考了以下优质资源：

五、本地运行步骤 🚀

1. 克隆仓库到本地：git clone [前端仓库地址]（将 [前端仓库地址] 替换为实际仓库链接）

2. 打开 Visual Studio Code，导入项目，安装依赖：npm install（确保已安装 Node.js 环境）

3. 修改接口基础路径：找到 src/utils/request.js（或 axios 配置文件），将 baseURL 改为后端服务的实际地址（如 http://localhost:8080）

4. 启动本地开发服务器：npm run dev

5. 项目启动成功后，根据终端提示的地址访问（默认通常为 http://localhost:5173）

六、项目结构 📂

src/
├── api/          // 接口请求封装（用户、商品、设备、动态等模块）
├── assets/       // 静态资源（图片、全局样式、图标等）
├── components/   // 公共组件（按钮、表单、卡片等可复用组件）
├── layouts/      // 布局组件（页面框架、导航栏、侧边栏等）
├── pages/        // 页面组件（各功能模块对应的页面）
│   ├── home/     // 首页相关
│   ├── mall/     // 家具商城相关
│   ├── smartHome/ // 智能家居控制相关
│   ├── social/   // 社交功能相关
│   └── user/     // 用户中心相关
├── router/       // 路由配置（页面路由映射、路由守卫）
├── store/        // 状态管理（Pinia 存储用户信息、购物车数据等）
├── utils/        // 工具函数（axios 封装、格式化工具等）
├── App.vue       // 根组件
└── main.js       // 入口文件
public/           // 公共静态资源
.env.development  // 开发环境配置
.env.production   // 生产环境配置
